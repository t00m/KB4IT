== Overview

This checklist covers baseline security hardening measures for Linux servers. It is intended as a starting point — every environment has specific requirements that may require additional controls or justify exceptions.

Apply these controls in a test environment and validate application functionality before deploying to production.

== System Updates

[cols="1,4", options="header"]
|===
| Item | Description
| OS packages | Apply all available security patches before first use
| Automatic updates | Enable for security-critical packages where operational risk permits
| Kernel | Reboot after kernel updates to activate the new version
|===

== User Accounts

[cols="1,4", options="header"]
|===
| Item | Description
| Root login | Disable direct root SSH login (`PermitRootLogin no` in `/etc/ssh/sshd_config`)
| Default accounts | Lock or remove unused system accounts
| Password policy | Enforce minimum length, complexity, and rotation via PAM
| Sudo | Grant sudo access only to named users; avoid `NOPASSWD` in production
| SSH keys | Prefer key-based authentication over passwords
|===

== Filesystem

[cols="1,4", options="header"]
|===
| Item | Description
| World-writable files | Audit with `find / -perm -002 -type f 2>/dev/null`
| SUID/SGID binaries | Audit with `find / -perm /6000 -type f 2>/dev/null`; remove where not required
| `/tmp` permissions | Ensure `/tmp` is mounted with `noexec,nosuid`
| Sensitive file permissions | `/etc/passwd` = `644`, `/etc/shadow` = `640`, `/etc/ssh/sshd_config` = `600`
|===

== Network

[cols="1,4", options="header"]
|===
| Item | Description
| Firewall | Enable and configure to allow only required ports
| Open ports | Audit with `ss -tlnp`; disable unused services
| IPv6 | Disable if not required (`net.ipv6.conf.all.disable_ipv6 = 1` in `/etc/sysctl.conf`)
| NTP | Configure time synchronisation; accurate timestamps are essential for log correlation
|===

== Services

[cols="1,4", options="header"]
|===
| Item | Description
| Unnecessary services | Disable with `systemctl disable --now <service>`
| Service users | Run services under dedicated, unprivileged accounts
| Default credentials | Change all default passwords immediately after installation
|===

== Logging and Auditing

[cols="1,4", options="header"]
|===
| Item | Description
| Syslog | Ensure logs are forwarded to a central, tamper-resistant system
| Audit daemon | Enable `auditd` and configure rules for privileged command execution
| Log retention | Define and enforce a retention policy
|===

== Notes

IMPORTANT: Security hardening is an ongoing process, not a one-time task. Re-audit after major configuration changes and review against current CIS benchmarks periodically.

WARNING: Aggressive hardening can break legitimate functionality. Test each change in isolation and document every exception with a justification.
